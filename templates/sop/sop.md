# Incident Response Procedure

## Overview
This document describes what to do when there is a production incident.

## Steps

1. Someone notices the incident and tells the team.
2. Figure out how bad it is.
3. Fix the problem.
4. Tell people what happened.

## Severity Levels
- High: system is down
- Medium: some features broken
- Low: minor issue

## Contact
Reach out to the on-call engineer if needed.
